# Stream Management:
1. 24hr per stream connection
2. Ping/Pong connection to keep the connection alive (20 seconds ping, 1 minute pong response)
3. Pong message contains exact Ping message payload. must send the frame as soon as possible after receiving the Ping frame.


# Websocket limitation: (Middleware)
1. 5 messages per second per connection.
3. 24hr per stream connection.
4. Ping/Pong connection to keep the connection alive (20 seconds ping, 1 minute pong response).
5. IPs that are repeatedly disconnecting or sending invalid messages may be banned.
6. A single connection can listen to a maximum of 1024 streams.
7. There is a limit of 300 connections per attempt every 5 minutes per IP.


# Shutdown Event to prevent data loss:
When the server is shutting down, it will send a shutdown event to all connected clients. The event will contain the following information:
{
  "stream": "!serverShutdown",
  "data": {
    "e": "serverShutdown", // Event type
    "E": 1770123456789     // Event time
  }
}

# Live Subscribing/Unsubscribing to streams:
1. To subscribe to a stream, the client must send a subscription request with the desired stream name(s) and any necessary parameters.
2. To unsubscribe from a stream, the client must send an unsubscription request with the desired stream name(s).
3. The server will respond with a confirmation message indicating whether the subscription or unsubscription was successful.
4. The ID is used to identify the subscription or unsubscription request and will be included in the confirmation message.
    a. "64bit integer ID" is generated by the client and must be unique for each request.
    b. "alphanumeric string ID" is generated. max length: 36
    c. "null". In the response, if the result received is null this means the request sent was a success for non-query requests (e.g. Subscribing/Unsubscribing).
